# Структура папок для изображений сервисов

## Автоматическая генерация путей

Теперь компоненты автоматически генерируют пути к изображениям на основе ID сервиса. Вам не нужно указывать пути в `services.json` - они формируются автоматически.

## Структура папок

Для каждого сервиса создайте папку с именем `service-{ID}` и положите туда изображения:

```
public/images/service/
├── service-1/          # Для сервиса с ID = 1
│   ├── 1.png          # Основное изображение
│   ├── 2.png          # Второе изображение
│   ├── 3.png          # Третье изображение
│   └── 4.png          # Четвертое изображение
├── service-2/          # Для сервиса с ID = 2
│   ├── 1.png
│   ├── 2.png
│   └── ...
├── service-3/          # Для сервиса с ID = 3
│   └── ...
└── ...
```

## Правила именования

1. **Папка**: `service-{ID}` где `{ID}` - это ID сервиса из `services.json`
   - Пример: `service-1`, `service-2`, `service-12`

2. **Файлы изображений**: `{номер}.{расширение}`
   - `1.png` - основное изображение (используется везде)
   - `2.png` - второе изображение
   - `3.png` - третье изображение
   - `4.png` - четвертое изображение

3. **Расширения**: поддерживаются `png`, `webp`, `jpg`, `jpeg`

## Примеры

### Сервис ID = 1 (Транскраниальный магнитный стимулятор)
```
public/images/service/service-1/
├── 1.png  ← Основное изображение
├── 2.png  ← Дополнительное
├── 3.png  ← Дополнительное
└── 4.png  ← Дополнительное
```

### Сервис ID = 2 (Аппарат для лечения мозговых сосудов)
```
public/images/service/service-2/
├── 1.png
├── 2.png
└── 3.png
```

## Как это работает

Компоненты автоматически формируют пути:
- **ID 1** → `/images/service/service-1/1.png`
- **ID 2** → `/images/service/service-2/1.png`
- **ID 3** → `/images/service/service-3/1.png`
- и т.д.

## Текущее состояние

✅ **Сервис ID 1**: Папка `serivce-1` (с опечаткой) уже существует с файлами:
- `1 (1).png`
- `1 (2).png`
- `1 (3).png`
- `1 (4).png`

⚠️ **Рекомендация**: Переименуйте папку `serivce-1` в `service-1` и переименуйте файлы:
- `1 (1).png` → `1.png`
- `1 (2).png` → `2.png`
- `1 (3).png` → `3.png`
- `1 (4).png` → `4.png`

## Для остальных сервисов

Создайте папки и положите изображения:
- `service-2/` для сервиса ID 2
- `service-3/` для сервиса ID 3
- `service-4/` для сервиса ID 4
- и т.д. до `service-12/`

## Fallback

Если изображение не найдено в папке сервиса, компоненты используют fallback изображения из `services.json` (если указаны) или стандартные placeholder.
